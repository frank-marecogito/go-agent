## 1. 日常检查

**执行时间**：每个工作日 09:30  
**执行人员**：值班工程师  
**预计耗时**：15 分钟

### 1.1 健康度检查

- [ ] **健康评分 > 80**
  - 查看 Grafana 仪表板
  - 如低于 80，查看 [01_monitoring_guide.md](./01_monitoring_guide.md) 第 3.2 节
  
- [ ] **无 P0/P1 告警**
  - 检查 Alertmanager
  - 检查 PagerDuty/值班群
  
- [ ] **系统可用性正常**
  - API 端点响应正常
  - 数据库连接正常
  - 向量数据库连接正常

### 1.2 监控指标检查

#### 1.2.1 存储层指标

- [ ] **存储总量** - 无异常增长
  ```promql
  memory_total_count
  # 期望：日增长 < 10%
  ```

- [ ] **修剪率 < 30%**
  ```promql
  memory_prune_rate
  # 期望：< 30%
  ```

- [ ] **存储延迟 P95 < 500ms**
  ```promql
  histogram_quantile(0.95, rate(memory_store_latency_seconds_bucket[5m]))
  # 期望：< 0.5s
  ```

#### 1.2.2 检索层指标

- [ ] **检索成功率 > 95%**
  ```promql
  memory_retrieve_success_rate
  # 期望：> 95%
  ```

- [ ] **空结果率 < 30%**
  ```promql
  memory_empty_result_rate
  # 期望：< 30%
  ```

- [ ] **检索延迟 P95 < 200ms**
  ```promql
  histogram_quantile(0.95, rate(memory_retrieve_latency_seconds_bucket[5m]))
  # 期望：< 0.2s
  ```

#### 1.2.3 质量层指标

- [ ] **去重率 < 20%**
  ```promql
  memory_dedup_rate
  # 期望：< 20%
  ```

- [ ] **重嵌入率 < 15%**
  ```promql
  memory_reembed_rate
  # 期望：< 15%
  ```

- [ ] **摘要覆盖率 > 50%**
  ```promql
  memory_summary_coverage
  # 期望：> 50%
  ```

#### 1.2.4 资源层指标

- [ ] **CPU 使用率 < 80%**
  ```promql
  node_cpu_usage_percent
  # 期望：< 80%
  ```

- [ ] **内存使用率 < 85%**
  ```promql
  node_memory_usage_percent
  # 期望：< 85%
  ```

- [ ] **磁盘使用率 < 80%**
  ```promql
  node_disk_usage_percent
  # 期望：< 80%
  ```

### 1.3 日志检查

- [ ] **无 ERROR 级别日志**
  ```bash
  # 查看最近 24 小时错误日志
  kubectl logs -l app=go-agent --since=24h | grep ERROR
  
  # 或使用日志系统
  grep "ERROR" /var/log/go-agent/*.log | tail -50
  ```

- [ ] **无频繁 WARN 日志**
  ```bash
  # 统计警告日志数量
  grep "WARN" /var/log/go-agent/*.log | wc -l
  # 期望：< 100 条/小时
  ```

- [ ] **关键错误已处理**
  - 数据库连接错误
  - 嵌入服务超时
  - 向量数据库错误

### 1.4 备份检查

- [ ] **昨日备份成功**
  - 检查备份日志
  - 验证备份文件大小
  - 确认备份存储位置

- [ ] **备份文件可访问**
  ```bash
  # 检查备份文件
  ls -lh /backup/memory/$(date -d yesterday +%Y%m%d)/
  
  # 验证备份完整性
  md5sum -c /backup/memory/$(date -d yesterday +%Y%m%d)/checksum.md5
  ```

## 2. 周维护

**执行时间**：每周一 10:00  
**执行人员**：运维工程师 + 后端工程师  
**预计耗时**：60 分钟

### 2.1 性能分析

- [ ] **查看性能趋势**
  ```promql
  # 检索延迟趋势（过去 7 天）
  histogram_quantile(0.95, rate(memory_retrieve_latency_seconds_bucket[1h]))
  
  # 存储延迟趋势（过去 7 天）
  histogram_quantile(0.95, rate(memory_store_latency_seconds_bucket[1h]))
  ```

- [ ] **识别性能瓶颈**
  - 延迟是否有上升趋势？
  - 是否有周期性峰值？
  - 是否与特定操作相关？

- [ ] **分析慢查询**
  ```sql
  -- 查询最慢的 10 次检索
  SELECT query, latency_ms, timestamp
  FROM retrieve_logs
  WHERE timestamp > NOW() - INTERVAL '7 days'
  ORDER BY latency_ms DESC
  LIMIT 10;
  ```

### 2.2 容量检查

- [ ] **存储容量趋势**
  ```promql
  # 存储增长趋势
  predict_linear(memory_total_size_gb[7d], 7*24*3600)
  
  # 预计多少天后满
  (disk_total_bytes - disk_used_bytes) / rate(disk_used_bytes[7d]) / 86400
  ```

- [ ] **评估扩容需求**
  - 当前容量使用率
  - 预计可用天数
  - 是否需要提前扩容

- [ ] **检查资源配额**
  ```bash
  # Kubernetes 资源使用
  kubectl top pods -l app=go-agent
  
  # 检查资源限制
  kubectl get deployment go-agent -o yaml | grep -A 10 resources
  ```

### 2.3 备份验证

- [ ] **随机抽样恢复测试**
  ```bash
  # 随机选择一个备份
  BACKUP_DATE=$(ls /backup/memory/ | shuf -n 1)
  
  # 恢复到测试环境
  pg_restore -d test_memory /backup/memory/$BACKUP_DATE/memory.dump
  
  # 验证数据完整性
  psql -d test_memory -c "SELECT COUNT(*) FROM memories;"
  ```

- [ ] **验证备份保留策略**
  ```bash
  # 检查备份文件数量
  ls -la /backup/memory/ | wc -l
  
  # 检查是否有超过保留期的备份
  find /backup/memory/ -type d -mtime +30
  ```

- [ ] **清理过期备份**
  ```bash
  # 删除 30 天前的备份
  find /backup/memory/ -type d -mtime +30 -exec rm -rf {} \;
  ```

### 2.4 参数审查

- [ ] **检查 MMR 参数**
  - 当前 Lambda 值
  - 各权重配置
  - 是否需要调整

- [ ] **查看参数变更记录**
  ```sql
  SELECT timestamp, lambda, changed_by, reason, ndcg_delta
  FROM mmr_params_log
  ORDER BY timestamp DESC
  LIMIT 10;
  ```

- [ ] **评估参数效果**
  - NDCG 是否有提升？
  - 用户满意度是否改善？
  - 是否有负面影响？

## 3. 月维护

**执行时间**：每月第一个周一 14:00  
**执行人员**：SRE 工程师 + 后端工程师 + 技术负责人  
**预计耗时**：120 分钟

### 3.1 健康度趋势分析

- [ ] **生成月度健康报告**
  ```promql
  # 月度健康评分趋势
  memory_health_score
  
  # 月度平均健康评分
  avg_over_time(memory_health_score[30d])
  ```

- [ ] **分析健康度变化**
  - 本月平均健康评分 vs 上月
  - 健康度最低点及原因
  - 健康度趋势（改善/恶化/稳定）

- [ ] **识别系统性问题**
  - 是否有重复出现的问题？
  - 是否有未解决的技术债务？
  - 是否需要架构调整？

### 3.2 参数调优评估

- [ ] **回顾本月参数变更**
  ```sql
  SELECT 
      DATE(timestamp) as date,
      lambda,
      similarity_weight,
      changed_by,
      reason,
      ROUND((after_ndcg - before_ndcg)::numeric, 4) as ndcg_delta
  FROM mmr_params_log
  WHERE timestamp > NOW() - INTERVAL '30 days'
  ORDER BY timestamp DESC;
  ```

- [ ] **评估调优效果**
  | 变更日期 | 变更内容 | NDCG 变化 | 用户满意度 | 评估 |
  |----------|----------|-----------|------------|------|
  | 03-05 | λ 0.7→0.75 | +0.02 | +0.1 | ✅ 有效 |
  | 03-12 | similarity 0.45→0.50 | -0.01 | -0.05 | ❌ 无效 |

- [ ] **制定下月调优计划**
  - 需要调整的参数
  - 预期目标
  - 测试计划

### 3.3 安全扫描

- [ ] **依赖漏洞扫描**
  ```bash

  # 或使用 nancy
  go list -json -m all | nancy sleuth
  ```

- [ ] **配置安全检查**
  - 敏感信息是否已加密？
  - API 密钥是否定期轮换？
  - 访问控制是否合理？

- [ ] **审计日志检查**
  ```sql
  -- 检查异常访问
  SELECT user_id, action, COUNT(*) as count
  FROM audit_logs
  WHERE timestamp > NOW() - INTERVAL '30 days'
  GROUP BY user_id, action
  HAVING COUNT(*) > 1000
  ORDER BY count DESC;
  ```
